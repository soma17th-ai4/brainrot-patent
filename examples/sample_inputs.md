# Sample Inputs

발표와 테스트에서 공통으로 사용할 입력입니다.

## 1순위 데모 입력

```text
방구로 가는 자동차
```

왜 좋은가:

- 황당함이 즉시 이해됩니다.
- 기술어 변환 효과가 잘 보입니다.
- 청구항이 웃기면서도 특허처럼 보이기 쉽습니다.

## 2순위 데모 입력

```text
기억을 USB처럼 꽂아서 옮기는 헬멧
```

왜 좋은가:

- SF/세계관 창작자 타깃과 잘 맞습니다.
- 뇌파, 저장장치, 신경 인터페이스 같은 검색 키워드로 확장하기 쉽습니다.

## 3순위 데모 입력

```text
잠을 대신 자주는 베개
```

왜 좋은가:

- 일반 사용자가 공감하기 쉽습니다.
- 수면, 생체신호, 자동 회복 같은 그럴듯한 용어로 바꾸기 좋습니다.

## 테스트 규칙

- Frontend, Backend, AI, Playwright는 최소 1순위 입력으로 동작을 확인합니다.
- 발표 전에는 3개 입력 모두 결과가 나오도록 확인합니다.
- 라이브 API가 실패하면 이 입력들의 사전 생성 결과를 fallback으로 사용합니다.
