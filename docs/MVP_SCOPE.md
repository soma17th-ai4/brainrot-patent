# MVP Scope

## 반드시 구현

1. 한국어 아이디어 입력
   - 입력 길이: 1~3문장
   - 영어 입력은 "한국어로 입력해주세요"로 안내
   - 위험 주제는 간단한 키워드 필터로 차단

2. 명세서 7개 섹션 생성
   - 발명의 명칭
   - 기술분야
   - 배경기술
   - 해결하려는 과제
   - 발명의 구성
   - 청구항
   - 요약

3. 청구항 생성
   - 독립항 1개
   - 종속항 3개
   - 문체는 특허 문서처럼 딱딱하게 유지

4. 결과 미리보기 UI
   - 입력 화면
   - 생성 중 상태
   - 결과 섹션 표시
   - 출처 또는 근거 표시 영역

5. 데모 안정화
   - 발표용 고정 입력 3개 준비
   - API 실패 시 샘플 결과 fallback 제공

## 가능하면 구현

- 섹션별 재생성
- PDF 다운로드
- 검색 결과 출처 URL 표시
- 특허 문서 같은 HTML 템플릿
- Playwright 기반 E2E 테스트

## 이번 2주에는 하지 않음

- 로그인, 회원가입
- 결제
- 실제 특허 출원 가능성 평가
- 진짜 신규성/진보성 판단
- 이미지 기반 도면 생성
- 다국어 지원
- 복잡한 벡터 DB 운영

## 우선순위

1순위는 "그럴듯한 특허 명세서 결과가 화면에 뜨는 것"입니다. PDF, RAG, 섹션 재생성은 결과 품질과 발표 안정성이 확보된 뒤 붙입니다.
